# 🗃️ Database Schema

Forkcast uses Supabase (Postgres). All primary keys are **UUIDs** (never Mongo `ObjectId`).

> Column names in Postgres use `snake_case`; the API layer transforms them to `camelCase` before returning JSON to the client. So `user_id` in the DB becomes `userId` in the response.

## `users`

| Column      | Type         | Notes                              |
|-------------|--------------|------------------------------------|
| `id`        | `uuid` PK    | Generated by the app (UUID v4)     |
| `username`  | `text`       | Unique                             |
| `password`  | `text`       | **bcrypt hash** — never plaintext  |
| `created_at`| `timestamptz`| Set on insert                      |

## `meals`

| Column           | Type         | Notes                                              |
|------------------|--------------|----------------------------------------------------|
| `id`             | `uuid` PK    |                                                    |
| `user_id`        | `uuid` FK    | → `users.id`                                       |
| `title`          | `text`       |                                                    |
| `ingredients`    | `text`       | Free-form (one per line)                           |
| `instructions`   | `text`       | Free-form                                          |
| `image_url`      | `text` null  | Cloudinary `secure_url`                            |
| `gallery_images` | `text` null  | JSON-encoded array of Cloudinary URLs              |
| `created_at`     | `timestamptz`|                                                    |
| `updated_at`     | `timestamptz`|                                                    |

## `meal_plans`

| Column       | Type         | Notes                                    |
|--------------|--------------|------------------------------------------|
| `id`         | `uuid` PK    |                                          |
| `user_id`    | `uuid` FK    | → `users.id`                             |
| `meal_id`    | `uuid` FK    | → `meals.id`                             |
| `planned_for`| `date`       | The day this meal is scheduled for       |
| `created_at` | `timestamptz`|                                          |

## Relationships

```
users (1) ────< meals
  │
  └────< meal_plans >──── meals
```

- A user has many meals (`meals.user_id → users.id`).
- A meal plan links a user to a meal for a given day.

See [operations/debugging.md](./debugging.md) for how to inspect and edit this data.
